## 1. Source artifacts and normalized spatial layers

Raw-source provenance is defined in [scripts/01_build_manifest.py](/Users/hq/code/immc-risk-intervention/scripts/01_build_manifest.py#L23), then normalized into processed geospatial layers.

Current source-to-layer mapping:

- OSM / Nominatim boundary and pan exports → [etosha_boundary.geojson](/Users/hq/code/immc-risk-intervention/data/processed/etosha_boundary.geojson), [pan_polygon.geojson](/Users/hq/code/immc-risk-intervention/data/processed/pan_polygon.geojson) via [scripts/02_normalize_boundary.py](/Users/hq/code/immc-risk-intervention/scripts/02_normalize_boundary.py#L17)
- Geofabrik roads extract and curated CSVs → [roads.geojson](/Users/hq/code/immc-risk-intervention/data/processed/roads.geojson), [tourist_roads.geojson](/Users/hq/code/immc-risk-intervention/data/processed/tourist_roads.geojson), [waterholes.geojson](/Users/hq/code/immc-risk-intervention/data/processed/waterholes.geojson), [camps.geojson](/Users/hq/code/immc-risk-intervention/data/processed/camps.geojson), [gates.geojson](/Users/hq/code/immc-risk-intervention/data/processed/gates.geojson) via [scripts/03_normalize_infrastructure.py](/Users/hq/code/immc-risk-intervention/scripts/03_normalize_infrastructure.py#L31)
- GBIF elephant occurrences → [elephant_density_points.parquet](/Users/hq/code/immc-risk-intervention/data/processed/elephant_density_points.parquet) via [scripts/04_normalize_wildlife.py](/Users/hq/code/immc-risk-intervention/scripts/04_normalize_wildlife.py#L52)
- Public lion and rhino figure images + GCP tables → digitized raw CSV detections via [scripts/04_digitize_reference_maps.py](/Users/hq/code/immc-risk-intervention/scripts/04_digitize_reference_maps.py#L131), then normalized into [lion_zones.geojson](/Users/hq/code/immc-risk-intervention/data/processed/lion_zones.geojson) and [rhino_reference_areas.geojson](/Users/hq/code/immc-risk-intervention/data/processed/rhino_reference_areas.geojson) via [scripts/04_normalize_wildlife.py](/Users/hq/code/immc-risk-intervention/scripts/04_normalize_wildlife.py#L85)
- NASA EONET wildfire events → [wildfire_history.geojson](/Users/hq/code/immc-risk-intervention/data/processed/wildfire_history.geojson) via [scripts/05_normalize_wildfires.py](/Users/hq/code/immc-risk-intervention/scripts/05_normalize_wildfires.py#L22)

## 2. Grid and per-cell spatial features

The analysis surface is a clipped 5 km grid built in UTM zone 33S by [scripts/06_build_grid.py](/Users/hq/code/immc-risk-intervention/scripts/06_build_grid.py#L26). It produces:

- [grid.geojson](/Users/hq/code/immc-risk-intervention/outputs/grid.geojson)
- [grid_centroids.geojson](/Users/hq/code/immc-risk-intervention/outputs/grid_centroids.geojson)

[scripts/07_build_features.py](/Users/hq/code/immc-risk-intervention/scripts/07_build_features.py#L100) derives per-cell features in [grid_features.parquet](/Users/hq/code/immc-risk-intervention/outputs/grid_features.parquet):

- distance to boundary / road / tourist road / gate / camp / waterhole / pan
- pan overlap area and ratio
- historical wildfire event count
- terrain class: `pan`, `pan_margin`, `boundary_edge`, `interior_savanna`

These features are the shared substrate for both the risk map and the optimization-prep layer.

## 3. Species-side surfaces

[scripts/08_build_species_layers.py](/Users/hq/code/immc-risk-intervention/scripts/08_build_species_layers.py#L140) writes [species_layers.parquet](/Users/hq/code/immc-risk-intervention/outputs/species_layers.parquet). Each grid cell gets four normalized wildlife-support surfaces:

- `elephant_density_norm`
  - KDE over processed elephant points ([scripts/08_build_species_layers.py](/Users/hq/code/immc-risk-intervention/scripts/08_build_species_layers.py#L97))
- `rhino_support_norm`
  - inside/buffer distance to rhino reference area
  - distance to waterholes
  - pan penalty ([scripts/08_build_species_layers.py](/Users/hq/code/immc-risk-intervention/scripts/08_build_species_layers.py#L104))
- `lion_support_norm`
  - inside/distance to lion support zones
  - pan penalty ([scripts/08_build_species_layers.py](/Users/hq/code/immc-risk-intervention/scripts/08_build_species_layers.py#L120))
- `herbivore_support_norm`
  - waterhole proximity
  - pan penalty
  - fire penalty
  - terrain bonus ([scripts/08_build_species_layers.py](/Users/hq/code/immc-risk-intervention/scripts/08_build_species_layers.py#L130))

## 4. Threat-side surfaces

[scripts/09_build_threat_layers.py](/Users/hq/code/immc-risk-intervention/scripts/09_build_threat_layers.py#L130) writes [threat_layers.parquet](/Users/hq/code/immc-risk-intervention/outputs/threat_layers.parquet). It builds three threat families.

### Poaching

Stored component columns:

- `poaching_gate_access`
- `poaching_boundary_access`
- `poaching_road_access`
- `poaching_tourist_road_access`
- `poaching_surveillance_gap`
- `poaching_rhino_value`

Formula in code:

`0.24*gate + 0.24*boundary + 0.18*road + 0.08*tourist_road + 0.08*surveillance_gap + 0.18*rhino`

Output column:

- `poaching_threat_norm`

Visual reference:

- [03_poaching_components.png](/Users/hq/code/immc-risk-intervention/outputs/risk_diagnostics/03_poaching_components.png)

### Wildfire

Stored component columns:

- `wildfire_recent_fire_suppression`
- `wildfire_flammable_land`
- `wildfire_water_remoteness`

Formula in code:

`wildfire=(0.60*flammable_land + 0.40*water_remoteness) * recent_fire_suppression`

Output column:

- `wildfire_threat_norm`

Visual reference:

- [04_wildfire_components.png](/Users/hq/code/immc-risk-intervention/outputs/risk_diagnostics/04_wildfire_components.png)

### Tourism

Stored component columns:

- `tourism_tourist_road_access`
- `tourism_camp_access`
- `tourism_gate_access`
- `tourism_waterhole_access`

Tourism is assembled in two steps:

- `tourism_pressure_norm`
  - `0.35*tourist_road + 0.25*camp + 0.15*gate + 0.25*waterhole`
- `tourism_threat_norm`
  - `tourism_pressure_norm * wildlife_presence`
  - where wildlife presence is `0.45*elephant + 0.25*lion + 0.30*herbivore` in [scripts/09_build_threat_layers.py](/Users/hq/code/immc-risk-intervention/scripts/09_build_threat_layers.py#L121)

Visual reference:

- [05_tourism_components.png](/Users/hq/code/immc-risk-intervention/outputs/risk_diagnostics/05_tourism_components.png)

## 5. From species surfaces and threat surfaces to risk

[scripts/10_build_risk_tensor.py](/Users/hq/code/immc-risk-intervention/scripts/10_build_risk_tensor.py#L70) merges species and threat outputs and constructs:

- `tensor[cell, species, threat] = species_presence_norm * threat_exposure_norm`

Stored artifact:

- [risk_tensor.npz](/Users/hq/code/immc-risk-intervention/outputs/risk_tensor.npz)

Threat-specific risk surfaces written into [composite_risk.geojson](/Users/hq/code/immc-risk-intervention/outputs/composite_risk.geojson):

- `poaching_risk_norm`
  - mean across all species for the poaching slice of the tensor
- `wildfire_risk_norm`
  - mean across all species for the wildfire slice
- `tourism_risk_norm`
  - mean across all species for the tourism-pressure slice

Composite heatmap:

- `composite_risk_norm`
  - weighted mean of threat-specific risks
  - weights in code:
    - `poaching = 1.0`
    - `wildfire = 1.5`
    - `tourism = 0.5`

## 6. Why these diagnostics fit the optimization-prep layer

The optimization-prep code is already active and downstream of the spatial pipeline:

- [scripts/13_build_surveillance_candidate_sites.py](/Users/hq/code/immc-risk-intervention/scripts/13_build_surveillance_candidate_sites.py#L66) consumes composite risk, grid centroids, and grid features
- [scripts/14_build_terrain_costs.py](/Users/hq/code/immc-risk-intervention/scripts/14_build_terrain_costs.py#L44) consumes grid features plus composite risk
- [scripts/15_build_surveillance_matrices.py](/Users/hq/code/immc-risk-intervention/scripts/15_build_surveillance_matrices.py#L34) consumes candidate sites plus terrain costs
- [scripts/_optimization_common.py](/Users/hq/code/immc-risk-intervention/scripts/_optimization_common.py#L338) validates the contract around `grid_features.parquet` and `composite_risk.geojson`

The new diagnostics do not change any of those files. They only read the current validated outputs and write separate explanatory PNGs into [outputs/risk_diagnostics](/Users/hq/code/immc-risk-intervention/outputs/risk_diagnostics).
